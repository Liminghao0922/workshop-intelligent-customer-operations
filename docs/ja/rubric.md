# 採点基準

合計: 100 点。

| 項目 | 点数 | 確認ポイント |
| --- | ---: | --- |
| リアルタイム音声設計 | 20 | 電話サービス、音声ゲートウェイ、Voice Live API、低遅延設計、有人引き継ぎ |
| 通話後分析 | 20 | ストレージトリガー、文字起こし、PII マスキング、要約、感情、意図、ダッシュボード |
| 実装可能性 | 15 | 実在する Azure サービス、明確なデータフロー、現実的な連携 |
| セキュリティとコンプライアンス | 15 | マネージド ID、RBAC、暗号化、PII 処理、保持期間、ネットワーク制御 |
| 信頼性と運用 | 10 | リトライ、冪等性、障害分離、監視、アラート |
| コストとパフォーマンス | 10 | リアルタイム経路とバッチ経路の分離、イベント駆動コンピュート、ストレージ階層 |
| 多言語設計 | 10 | 英語、日本語、中国語が音声、プロンプト、ナレッジ、分析に反映されている |

## よくあるミス

- 重い分析処理をリアルタイム通話経路に入れる。
- 生の PII を直接生成 AI 分析に送る。
- 図に「AI」とだけ書き、電話連携、保存、トリガー、監視、CRM 引き継ぎがない。
- ダッシュボードと運用テレメトリを忘れる。
- 多言語対応を UI 翻訳だけと考える。
